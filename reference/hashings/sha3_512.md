# SHA3-512哈希算法 (hashings.sha3_512)

SHA3-512是SHA-3家族的一员，产生512位（64字节）的哈希值。基于Keccak算法，提供最高的安全边际。

## 特性

- **摘要长度**：512位（64字节）
- **块大小**：576位（72字节）- 注意与SHA3-256不同
- **输出格式**：通常表示为128个十六进制字符
- **安全性**：极高，抗所有已知攻击

## 使用方法

```lua
local sha3_512 = require 'hashings.sha3_512'

-- 创建SHA3-512哈希对象并计算哈希
local h = sha3_512:new('Hello World')
print(h:hexdigest())  -- 输出: 128位十六进制字符串

-- 流式处理
local h = sha3_512:new()
h:update('First part ')
h:update('Second part')
print(h:hexdigest())
```

## 接口说明

### new

创建SHA3-512哈希对象

#### 函数原型

```lua
function sha3_512:new(data)
end
```

#### 参数说明

* data
  可选，初始数据字符串

#### 返回值

返回SHA3-512哈希对象

### update

更新哈希计算状态

#### 函数原型

```lua
function sha3_512:update(data)
end
```

#### 参数说明

* data
  要添加到哈希计算中的数据字符串

### digest

获取二进制哈希摘要

#### 函数原型

```lua
function sha3_512:digest()
end
```

#### 返回值

返回64字节的二进制字符串

### hexdigest

获取十六进制哈希摘要

#### 函数原型

```lua
function sha3_512:hexdigest()
end
```

#### 返回值

返回128个字符的十六进制字符串（大写）

### copy

复制哈希对象及其状态

#### 函数原型

```lua
function sha3_512:copy()
end
```

#### 返回值

返回包含相同内部状态的新SHA3-512对象

## 性能特点

- **计算速度**：比SHA-512快（SHA-3的效率优势）
- **内存占用**：比SHA-512低（更小的状态）
- **安全性**：极高，512位提供巨大安全边际

## 类属性

```lua
sha3_512.digest_size  -- 64 (字节)

```

## SHA3-512 vs SHA-512

虽然摘要长度相同，但算法完全不同：

| 特性 | SHA3-512 | SHA-512 |
| :--- | :--- | :--- |
| 算法 | Keccak | SHA-2家族 |
| 结构 | 海绵结构 | Merkle-Damgård |
| 抗侧信道 | 是 | 否 |
| 抗长度扩展 | 是 | 否 |
| 初始化 | 参数化不同 | IV不同 |
| 性能 | 通常更快 | 稍慢 |

| 性能敏感 | SHA3-256 | 块更大但摘要更短 |

## 完整示例

### 密钥派生

```lua
local sha3_512 = require 'hashings.sha3_512'

-- 从主密钥派生多个子密钥
local function derive_keys(master_key, contexts)
    local derived = {}

    for _, context in ipairs(contexts) do
        local h = sha3_512:new()
        h:update(master_key)
        h:update(context)
        derived[context] = h:hexdigest()
    end

    return derived
end

local keys = derive_keys('super_secret_master_key', {
    'encryption_key',
    'signing_key',
    'mac_key'
})
```

### 文件指纹

```lua
local sha3_512 = require 'hashings.sha3_512'

local function file_fingerprint(filename)
    local file = io.open(filename, 'rb')
    if not file then return nil, 'Cannot open file' end

    local h = sha3_512:new()
    local file_size = 0

    while true do
        local block = file:read(128 * 1024)  -- 128KB块
        if not block then break end
        h:update(block)
        file_size = file_size + #block
    end
    file:close()

    local fingerprint = h:hexdigest()
    return fingerprint, file_size
end

local fp, size = file_fingerprint('large_file.bin')
print(string.format('SHA3-512: %s, Size: %d bytes', fp, size))
```

## 安全优势

### 抗量子计算

SHA-3的设计考虑了抗量子计算攻击的能力，相比SHA-2具有更好的后量子安全性。

### 抗碰撞

512位输出使得暴力破解搜索空间为2^512，这在可预见的未来是不可能的。

### 海绵结构优势

- **可调参数**：可以根据需求调整容量
- **简单安全**：算法结构更简单，便于分析
- **灵活性**：相同基础可以产生不同输出长度

## 与SHA-256/SHA-512组合使用

在一些应用中，可以组合使用多种哈希算法：

```lua
local sha3_256 = require 'hashings.sha3_256'
local sha3_512 = require 'hashings.sha3_512'

local function double_hash(data)
    local h1 = sha3_256:new(data)
    local hash1 = h1:digest()

    local h2 = sha3_512:new(hash1)
    return h2:hexdigest()
end

-- 或使用SHA-256和SHA-512提供两个不同的哈希
local function dual_hash(data)
    return {
        sha256 = sha3_256:new(data):hexdigest(),
        sha512 = sha3_512:new(data):hexdigest()
    }
end
```

## 性能比较

### 相同摘要长度对比

对于256位输出：
- **SHA3-256**：32字节输出，136字节块
- **SHA-256**：32字节输出，64字节块

SHA3-256在处理大数据时可能因为更大的块大小而更高效。

### 内存占用

SHA3-512的状态比SHA-512小，使得它在资源受限环境中更有优势。

## 最佳实践

1. **新项目**：优先考虑SHA-3而非SHA-2
2. **长期安全**：对于需要长期保护的敏感数据
3. **标准检查**：确认应用标准是否允许SHA-3
4. **性能测试**：在实际环境中测试性能差异

## 迁移示例

### 从SHA-512迁移

```lua
-- 之前
local sha512 = require 'hashings.sha512'
local h = sha512:new(data)

-- 之后（接口完全相同）
local sha3_512 = require 'hashings.sha3_512'
local h = sha3_512:new(data)

-- 输出长度相同（64字节），只需更改模块名
```

### 混合使用

```lua
-- 逐步迁移：同时计算两种哈希
local sha512 = require 'hashings.sha512'
local sha3_512 = require 'hashings.sha3_512'

local function migrate_hash(data)
    return {
        old = sha512:new(data):hexdigest(),
        new = sha3_512:new(data):hexdigest()
    }
end
```
