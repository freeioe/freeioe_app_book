# SHA3-256哈希算法 (hashings.sha3_256)

SHA3-256是SHA-3家族的一员，产生256位（32字节）的哈希值。基于Keccak算法，是NIST在2015年标准化的安全哈希算法。

## 特性

- **摘要长度**：256位（32字节）
- **块大小**：1088位（136字节）- Keccak的r+c
- **输出格式**：通常表示为64个十六进制字符
- **安全性**：高，SHA-3竞赛获胜者，抗侧信道攻击

## 与SHA-256的区别

SHA-3（Keccak）与SHA-2（包括SHA-256）是完全不同的设计：
- **SHA-2**：基于Merkle-Damgård构造
- **SHA-3**：基于海绵结构（sponge construction）

## 使用方法

```lua
local sha3_256 = require 'hashings.sha3_256'

-- 创建SHA3-256哈希对象并计算哈希
local h = sha3_256:new('Hello World')
print(h:hexdigest())  -- 输出: 64位十六进制字符串

-- 流式处理
local h = sha3_256:new()
h:update('First part ')
h:update('Second part')
print(h:hexdigest())
```

## 接口说明

### new

创建SHA3-256哈希对象

#### 函数原型

```lua
function sha3_256:new(data)
end
```

#### 参数说明

* data
  可选，初始数据字符串

#### 返回值

返回SHA3-256哈希对象

### update

更新哈希计算状态

#### 函数原型

```lua
function sha3_256:update(data)
end
```

#### 参数说明

* data
  要添加到哈希计算中的数据字符串

### digest

获取二进制哈希摘要

#### 函数原型

```lua
function sha3_256:digest()
end
```

#### 返回值

返回32字节的二进制字符串

### hexdigest

获取十六进制哈希摘要

#### 函数原型

```lua
function sha3_256:hexdigest()
end
```

#### 返回值

返回64个字符的十六进制字符串（大写）

### copy

复制哈希对象及其状态

#### 函数原型

```lua
function sha3_256:copy()
end
```

#### 返回值

返回包含相同内部状态的新SHA3-256对象

## 使用示例

### 数据完整性验证

```lua
local sha3_256 = require 'hashings.sha3_256'

local function verify_data(data, expected_hash)
    local h = sha3_256:new(data)
    local calculated_hash = h:hexdigest()

    return calculated_hash == expected_hash
end

local data = 'important data'
local hash = sha3_256:new(data):hexdigest()

if verify_data(data, hash) then
    print('Data integrity verified')
end
```

### 密码存储（带盐）

```lua
local sha3_256 = require 'hashings.sha3_256'

local function hash_password(password, salt)
    local h = sha3_256:new()
    h:update(salt)
    h:update(password)
    return h:hexdigest()
end

local salt = 'random_salt_value'
local hashed = hash_password('user_password', salt)
```

### 数字签名准备

```lua
local sha3_256 = require 'hashings.sha3_256'

local function prepare_signature(data, private_key)
    local h = sha3_256:new()
    h:update(data)
    h:update(private_key)
    return h:hexdigest()
end
```

## 性能特点

- **计算速度**：比SHA-256稍慢
- **内存占用**：比SHA-256稍高（更大的块大小）
- **安全性**：极高，SHA-3抗侧信道攻击

## 类属性

```lua
sha3_256.digest_size  -- 32 (字节)
sha3_256.block_size   -- 136 (字节)
```

## SHA-3家族对比

| 算法 | 摘要长度 | 块大小 | 适用场景 |
| :--- | :--- | :--- | :--- |
| SHA3-224 | 224位 | 144字节 | 需要较短摘要 |
| **SHA3-256** | **256位** | **136字节** | **通用安全哈希** |
| SHA3-384 | 384位 | 104字节 | 需要更长摘要 |
| SHA3-512 | 512位 | 72字节 | 最高安全需求 |

## 使用建议

### 适用场景

✅ **推荐用于**：
- 新项目或新实现
- 需要抗侧信道攻击的场景
- 对安全性有高要求的应用
- 未来兼容性考虑

### 不适用场景

❌ **不推荐用于**：
- 需要极高性能的场景（使用BLAKE2）
- 内存受限的嵌入式设备（使用SHA-256）

## Keccak基础

SHA3-256基于Keccak算法，底层使用Keccak-f[1600]排列：

```lua
local keccak = require 'hashings.keccak'

-- SHA3-256使用特定的参数
-- rate (r) = 1088
-- capacity (c) = 512
-- digest_size = 256
```

这些参数确保了输出的安全性和抗碰撞性。

## 应用场景

### 区块链

```lua
local sha3_256 = require 'hashings.sha3_256'

-- 一些区块链使用SHA3-256
local function hash_transaction(from, to, amount)
    local h = sha3_256:new()
    h:update(from .. to .. tostring(amount))
    return h:hexdigest()
end
```

### 文件校验

```lua
local function file_sha3_256(filename)
    local file = io.open(filename, 'rb')
    if not file then return nil end

    local h = sha3_256:new()
    while true do
        local block = file:read(64 * 1024)  -- 64KB块
        if not block then break end
        h:update(block)
    end
    file:close()

    return h:hexdigest()
end
```

## 安全优势

### 抗长度扩展攻击

SHA-3的海绵结构天然抗长度扩展攻击，不需要像MD5/SHA-1那样特殊的填充处理。

### 抗碰撞

SHA-3的设计目标包括抗碰撞性，目前没有已知的有效碰撞攻击。

### 抗侧信道

Keccak排列的抗侧信道特性使得SHA-3在硬件实现时更安全。

## 最佳实践

1. **新项目首选**：对于新项目，SHA-3比SHA-2更安全
2. **密码哈希**：仍然建议使用专门的密码哈希如PBKDF2、bcrypt
3. **性能权衡**：如果性能是关键，BLAKE2可能是更好的选择
4. **标准兼容**：某些标准可能要求使用SHA-2而非SHA-3

## 迁移建议

### 从SHA-256迁移

```lua
-- 之前
local sha256 = require 'hashings.sha256'

-- 之后
local sha3_256 = require 'hashings.sha3_256'
```

迁移简单，接口完全一致，只需更改模块名。
