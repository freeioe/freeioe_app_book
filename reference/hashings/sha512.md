# SHA-512哈希算法 (hashings.sha512)

SHA-512（Secure Hash Algorithm 512-bit）是SHA-2家族的成员，产生512位（64字节）的哈希值。它使用64位运算，在64位系统上性能优异。

## 特性

- **摘要长度**：512位（64字节）
- **块大小**：1024位（128字节）
- **输出格式**：通常表示为128个十六进制字符
- **安全性**：目前认为安全，无已知实用碰撞攻击
- **性能**：在64位系统上可能比SHA-256更快

## 使用方法

```lua
local sha512 = require 'hashings.sha512'

-- 创建SHA-512哈希对象并计算哈希
local h = sha512:new('Hello World')
print(h:hexdigest())  -- 输出: 2C74FD17EDAFD80D8447B0D46741EE243B7EB74DD2149A0AB1B9246FB30382F27E853D8585719E0E67CBDA0DAA87F6C7B4...

-- 流式处理
local h = sha512:new()
h:update('First part ')
h:update('Second part')
print(h:hexdigest())

-- 获取二进制输出
local h = sha512:new('Hello World')
local binary = h:digest()  -- 64字节二进制数据
```

## 接口说明

### new

创建SHA-512哈希对象

#### 函数原型

```lua
function sha512:new(data)
end
```

#### 参数说明

* data
  可选，初始数据字符串

#### 返回值

返回SHA-512哈希对象

### update

更新哈希计算状态

#### 函数原型

```lua
function sha512:update(data)
end
```

#### 参数说明

* data
  要添加到哈希计算中的数据字符串

### digest

获取二进制哈希摘要

#### 函数原型

```lua
function sha512:digest()
end
```

#### 返回值

返回64字节的二进制字符串

### hexdigest

获取十六进制哈希摘要

#### 函数原型

```lua
function sha512:hexdigest()
end
```

#### 返回值

返回128个字符的十六进制字符串（大写）

### copy

复制哈希对象及其状态

#### 函数原型

```lua
function sha512:copy()
end
```

#### 返回值

返回包含相同内部状态的新SHA-512对象

## 使用示例

### 大文件校验和

```lua
local sha512 = require 'hashings.sha512'

local function file_sha512(filename)
    local file = io.open(filename, 'rb')
    if not file then return nil, 'Cannot open file' end

    local h = sha512:new()
    while true do
        local block = file:read(128 * 1024)  -- 128KB块
        if not block then break end
        h:update(block)
    end
    file:close()
    return h:hexdigest()
end

local hash = file_sha512('large_file.iso')
print('SHA-512:', hash)
```

### 高安全性场景

```lua
local sha512 = require 'hashings.sha512'

-- 用于需要更高安全性的场景
local function high_security_hash(data)
    local h = sha512:new()
    h:update(data)
    return h:hexdigest()
end
```

### 密钥派生（配合HMAC）

```lua
local sha512 = require 'hashings.sha512'
local hmac = require 'hashings.hmac'

-- 创建HMAC-SHA512
local auth = hmac:new(sha512, 'secret_key', 'message_to_authenticate')
print(auth:hexdigest())
```

## 性能特点

- **计算速度**：
  - 64位系统：快（得益于64位运算）
  - 32位系统：较慢
- **内存占用**：较高（128字节缓冲区 + 8个64位状态变量）
- **安全性**：极高，512位提供更强的抗碰撞性

## 类属性

```lua
sha512.digest_size  -- 64 (字节)
sha512.block_size   -- 128 (字节)
```

## SHA-256 vs SHA-512

### 选择SHA-512的场景

✅ **64位系统**：性能可能优于SHA-256
✅ **更高安全性要求**：512位提供更大的安全边际
✅ **需要更长摘要**：某些应用需要更长哈希值
✅ **未来兼容性**：更长的摘要为未来留出安全空间

### 选择SHA-256的场景

✅ **32位系统**：性能优于SHA-512
✅ **存储/带宽受限**：256位摘要更短
✅ **通用用途**：256位已足够安全

## 安全应用场景

### 数字签名

```lua
local sha512 = require 'hashings.sha512'

-- 对重要文档进行签名准备
local function document_hash(document_content)
    local h = sha512:new()
    h:update(document_content)
    return h:hexdigest()
end
```

### 密码哈希（多重迭代）

```lua
local sha512 = require 'hashings.sha512'

local function secure_password_hash(password, salt, iterations)
    local h = sha512:new()
    h:update(salt)
    h:update(password)

    local result = h:digest()

    -- 多次迭代增加安全性
    for i = 2, iterations do
        h = sha512:new(result)
        h:update(salt)
        result = h:digest()
    end

    return h:hexdigest()
end

-- 注意：生产环境应使用PBKDF2、bcrypt或scrypt
local hashed = secure_password_hash('password', 'random_salt', 10000)
```

### 区块链应用

```lua
local sha512 = require 'hashings.sha512'

-- 区块链中的Merkle树节点哈希
local function merkle_hash(left_hash, right_hash)
    local h = sha512:new()
    h:update(left_hash)
    h:update(right_hash)
    return h:hexdigest()
end
```

## 实现细节

SHA-512与SHA-256的主要区别：

1. **摘要长度**：512位 vs 256位
2. **块大小**：1024位 vs 512位
3. **轮数**：80轮 vs 64轮
4. **字长**：64位运算 vs 32位运算
5. **初始值和常数**：不同的初始化参数

这些差异使SHA-512在64位系统上具有性能优势，同时提供更长的哈希值。

## 迁移指南

### 从SHA-256迁移

```lua
-- 之前
local sha256 = require 'hashings.sha256'
local h = sha256:new(data)
local hash = h:hexdigest()  -- 64字符

-- 之后
local sha512 = require 'hashings.sha512'
local h = sha512:new(data)
local hash = h:hexdigest()  -- 128字符
```

注意事项：
- 存储空间需求加倍
- 网络传输开销增加
- 确保系统支持存储更长的哈希值
